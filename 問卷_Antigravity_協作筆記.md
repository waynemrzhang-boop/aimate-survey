## 六、survey-data.js 格式速查（未來的你／Claude 看這段就懂）

三大區塊：
- `window.SURVEY_LANDING_DATA`：首頁文字（badge／title／subtitle／roles）。
- `window.SURVEY_WELCOME_TEXTS`：家長版／學生版的開頭語。
- `window.SURVEY_QUESTIONS`：題庫，分 `parent: []` 與 `student: []` 兩個陣列。

每一題（物件）的欄位：

| 欄位 | 意思 |
|---|---|
| `id` | 唯一代號（如 `q1`）。**新增題請用不衝突的代號**（如 `q_free`），才不會打亂後台圖表對應 |
| `title` | 題目文字 |
| `type` | `single`（單選）／`multiple`（複選）／`email_signup`（留 email）／`textarea`（簡答） |
| `badge` | 分類標籤（畫面上的小標） |
| `options` | 選項陣列（textarea 不用） |
| `intro` | 題目前的一段說明文字（單選題已驗證可用；`email_signup` 可能不吃，要實測） |
| `filter` | `(ans) => true` 時直接結束問卷（用來篩掉非目標填答者） |
| `hasOther` | `true` 會多一個「其他＿＿」可填欄 |
| `triggerEffect` | `(ans) => true` 時觸發星星慶祝特效（學生版用） |

顯示順序是看**陣列裡的排列順序**，不是看 id；所以新增題只要放對位置即可，id 不必連號。

## 八、做問卷的設計原則（讓數據有公信力，當「憑據」用）

- **先量、後賣**：前段用中性問法老實測量（＝你的憑據），後段才放 pitch ＋ 收 email。
- **別誘導**：不要用「你覺得有什麼好處？」「只要花…」這種幫對方想好答案、帶風向的問法。
- **問過去行為 > 問未來意願**：問「現在每月花多少補習」勝過「你會不會付」。
- **價格給具體區間**，且區間要跨過你的定價，才看得出誰願意付到那。
- 每個「選好處／選賣點」題都放一個**逃生口**（例如「其實沒什麼用」），不然全是好話會灌水。
- **最強憑據 ＝ 願意留 email 試用的人數**（嘴巴說有興趣不算）。
- 樣本別只發親友（會客氣說好），盡量發給**陌生但對的人**（家長社團、班群）。

## 九、目前的產品定位（問卷在驗證的東西）

- **免費**：幫你整理各科考卷（拍照存檔、自動分類）——**所有科目**，這是把人拉進來的鉤子。
- **付費進階**：AI 針對錯題逐題追問 ＋ 系統性統整弱點——**先做數學**，其他科之後陸續加。
- **訂價方向**：每月約 NT$100–200（先前抓 150）。
- 鐵律：免費＝獲客，付費 AI＝收入；兩條軸分開，付費牆只加在「AI 追問」那關。
